# microsoft/agent-framework PR Summary — 2026-05-11

## Breaking Changes

### ⚠️ [#5752](https://github.com/microsoft/agent-framework/pull/5752) — Python: Migrate `agent-framework-a2a` to `a2a-sdk` v1.0 *(giles17)*

The Python `agent-framework-a2a` package has been migrated from `a2a-sdk` v0.3.x to **v1.0**, aligning with the A2A protocol v1.0 release. The v1.0 API surface is **not backwards-compatible** with v0.3.x. Mirrors the .NET migration in [#5423](https://github.com/microsoft/agent-framework/pull/5423). Closes [#5661](https://github.com/microsoft/agent-framework/issues/5661).

> **Action required:** Update all imports and usages of `agent-framework-a2a` to conform to the `a2a-sdk` v1.0 API.

### ⚠️ [#5753](https://github.com/microsoft/agent-framework/pull/5753) — .NET: Remove Foundry Toolbox Server-Side Tools Support *(alliscode)*

Removes the ability to pass Foundry Toolbox tools as server-side Responses API tools. Mirrors Python cleanup in [#5671](https://github.com/microsoft/agent-framework/pull/5671). The supported path going forward is the **hosted-agent MCP toolbox** route (`HostedMcpToolboxAITool` + `FoundryToolboxService`). Consumers using the removed pattern must migrate.

## Major Updates

### [#5739](https://github.com/microsoft/agent-framework/pull/5739) — .NET: Configurable Access Controls for the DevUI HTTP Surface *(moonbox3)*

`Microsoft.Agents.AI.DevUI` now ships with built-in, configurable access controls for its HTTP endpoints (`/devui`, `/v1/entities`, and related routes). Previously these development-time endpoints were fully open with no configurability, risking accidental exposure in shared or non-local environments.

### [#4953](https://github.com/microsoft/agent-framework/pull/4953) — .NET: Support Reasoning Events in AGUI *(jeffinsibycoremont)*

Adds **7 new event types** for reasoning in the AGUI protocol layer, closing a long-standing gap tracked in [#2619](https://github.com/microsoft/agent-framework/issues/2619) / [#2558](https://github.com/microsoft/agent-framework/issues/2558). .NET consumers using AGUI streaming can now receive and handle model reasoning traces alongside standard response events.

### [#5679](https://github.com/microsoft/agent-framework/pull/5679) — .NET: Add `IChatMessageInjector` for Message Injection During Function Loop *(westey-m)*

Introduces a new `IChatMessageInjector` interface on `PerServiceCallChatHistoryPersistingChatClient`. Enables tool code to **enqueue follow-up user messages** that the model processes automatically within the same function execution loop — unlocking multi-turn agentic scenarios driven by tool results without external orchestration.

### [#5701](https://github.com/microsoft/agent-framework/pull/5701) — .NET: Hosted Agents RAG Sample with Azure AI Search *(rogerbarreto)*

Adds a complete **RAG sample** for Hosted Agents backed by **Azure AI Search**, plus a live `Foundry.Hosting` integration test with **canary-token assertions** proving the model grounded its answer in retrieved content rather than training data. Closes [#5693](https://github.com/microsoft/agent-framework/issues/5693).

### [#5698](https://github.com/microsoft/agent-framework/pull/5698) — .NET: Hosted-Files Sample + `AgentSessionFiles` SDK Companion + Integration Test *(rogerbarreto)*

Delivers a .NET parallel of the Python `06_files` Hosted Agents sample, a code-first companion exercising the alpha `Azure.AI.Projects.AgentSessionFiles` SDK, and an end-to-end integration test in `Foundry.Hosting`. Closes [#5691](https://github.com/microsoft/agent-framework/issues/5691).
